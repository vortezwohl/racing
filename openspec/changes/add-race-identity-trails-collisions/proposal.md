## Why

当前比赛内载具缺少稳定可读的身份识别、尾流反馈和实体碰撞反馈，玩家难以判断 NPC 位置与交互关系；同时现有载具与 CPU 存在性能差异，不符合公平竞速的预期。

本变更将把“外观差异”和“比赛性能”拆开：载具可以拥有不同模型、体积、名字颜色和尾流颜色，但基础性能必须一致，并通过可验证的尾流加速、碰撞减速和动态视角增强速度感。

## What Changes

- 为比赛中的每个非本地玩家载具显示头顶名字与同色三角指示器。
- 为每台载具分配稳定的身份颜色，名字、三角指示器和尾流颜色保持一致。
- 修正菜单载具名字视觉大小不一致的根因，确保名称字体与显示尺寸稳定一致。
- 为载具尾部增加短距离 TRON 风格尾流，尾流长度随速度增加但有统一上限。
- 增加“吃尾流”机制：进入其他载具尾流时逐步积累加速增益，离开后平滑衰减回正常性能。
- 增加载具之间的实体碰撞处理：载具不可互相穿越，碰撞触发短暂减速，并根据相对速度产生挤压偏移。
- 保持 hitbox 在正式比赛中完全透明，仅允许 debug 模式可视化。
- 为玩家相机增加速度驱动的动态 FOV，静止时较窄，高速和尾流增益时逐步变宽。
- 统一所有载具基础性能参数，并移除 CPU 随机性能差异，确保公平性。

## Capabilities

### New Capabilities

- `race-vehicle-identity`: 定义比赛中载具身份颜色、头顶名字、三角指示器与菜单名称一致性要求。
- `race-draft-trails`: 定义载具尾流视觉、尾流可被吃到、尾流加速累积与衰减规则。
- `race-vehicle-collisions`: 定义载具 hitbox、载具间不可穿越、碰撞减速与挤压偏移规则。
- `race-camera-speed-fov`: 定义玩家相机 FOV 随速度变化的规则。
- `race-performance-fairness`: 定义所有载具基础性能一致、CPU 不得随机获得性能差异的公平性规则。

### Modified Capabilities

- 无。

## Impact

- 主要影响 `src/scenes/GameScene.ts`、`src/objects/Vehicle.ts`、`src/objects/Player.ts`、`src/objects/CPU.ts`、`src/utils/interfaces.ts` 与 `data/vehicles/*.ts`。
- 可能新增小型系统类或工具函数，用于身份标记、尾流采样、尾流增益、载具碰撞和公平性能配置。
- 不引入新的运行时依赖，不接入外部物理引擎。
- 需要保留 debug 模式下的可观察性，同时保证正式模式 hitbox 完全不可见。
