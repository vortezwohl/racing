## Why

当前比赛页的 HUD、设置入口和结算页在视觉语言上与菜单页明显脱节：HUD 仍然是简单 DOM 白字排版，缺少统一的科幻霓虹风格；比赛中没有可点击的设置入口；结算页则以近乎整屏黑场和临时占位式面板呈现，既破坏赛道画面的连续观感，也无法承载更多比赛结果信息。

现在需要把这些比赛内 UI 一次性升级为统一的 Canvas 风格界面，因为单页比赛壳体和随机发车位会话已经具备了重进基础，继续保留当前分散且粗糙的比赛 UI 会让整体品质和可扩展性明显落后于现有菜单与赛道表现。

## What Changes

- 将比赛 HUD 从当前 DOM 文本更新模式重构为基于 Canvas 的统一绘制层，重新布局为左上显示圈数和名次、中上显示计时、右上显示 `KM/H` 时速，并统一采用 Tron 风格的蓝色复古科幻视觉。
- 在比赛页右上角 HUD 上方新增常驻设置按钮，使用 Canvas 绘制小齿轮图标，并支持点击打开非暂停式设置弹窗。
- 在设置弹窗中提供 `重新开始比赛` 与 `退出到菜单` 两个操作，并要求这两条路径都通过新的干净比赛会话重建流程执行。
- 将当前结算黑屏面板重构为保留赛道背景的半透明结果面板，直接叠加在比赛页之上，不再依赖整页黑场来承载结算信息。
- 在结算面板中展示平均时速、每圈耗时、玩家最终名次、已冲线 NPC 的结果更新，以及按名字和颜色展示的本场排行榜。
- 在结算状态下保留右上角设置按钮位置不变，并在结算面板底部新增可点击的 `BACK` 与 `RETRY` 文本操作。
- 明确约束所有重新开始比赛的入口都必须创建新的比赛会话，并重新随机化玩家与 NPC 的发车位分配。

## Capabilities

### New Capabilities
- `race-canvas-hud`: 使用专用 Canvas HUD 层渲染比赛核心信息、倒计时、科幻字体样式与布局，而不是继续依赖分散的 DOM 文本节点。
- `in-race-settings-overlay`: 在比赛内提供常驻齿轮设置按钮和非暂停式操作弹窗，支持回到菜单与重新开始比赛。
- `race-results-overlay`: 在比赛结束后以半透明叠加面板展示完整结算信息、排行榜和结算操作，而不是使用黑场占位页。

### Modified Capabilities
- `scene-lifecycle-and-race-reentry`: 扩展比赛内设置弹窗、结算面板与重开/退出路径下的会话重建要求，确保不同入口都能干净地返回菜单或创建新比赛会话。
- `race-five-vehicle-start-grid`: 补充重新开始比赛场景下的随机发车位要求，确保每次通过设置弹窗或结算面板重开时都会重新随机分配发车位。

  - `src/objects/Track.ts`
  - `src/objects/Vehicle.ts`

  - 结算信息聚合与展示

- 依赖影响：
  - 不引入新的前端框架
